# Finish Branch

Close the loop after verification.

## Process

1. Summarize what changed in user-facing terms.
2. List any remaining risks, TODOs, or follow-up decisions.
3. Recommend the next integration step: keep iterating, commit locally, or prepare a PR.
4. If git is available, suggest focused commit scopes.
5. If git is not available, say so and keep the wrap-up non-git-specific.

## Rules

- Do not hide open risks.
- Keep the wrap-up short and actionable.
- Do not propose merge steps before verification is complete.
